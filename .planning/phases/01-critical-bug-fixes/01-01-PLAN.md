<task type="auto">
  <name>Task 1: 删除 db.js 中的破坏性 DELETE 语句（BUG-01）</name>
  <files>db.js</files>
  <action>
    定位 db.js 第 243-253 行的 db.exec() 调用块，将其替换为仅保留幂等的 CREATE UNIQUE INDEX 操作。
    
    **删除**以下完整块（包含 DELETE 语句）：
    ```js
    db.exec(`
      DELETE FROM source_channels
      WHERE id NOT IN (
        SELECT MIN(id)
        FROM source_channels
        GROUP BY user_id, url
      );
    
      CREATE UNIQUE INDEX IF NOT EXISTS idx_source_channels_user_url
      ON source_channels(user_id, url);
    `);
    ```
    
    **替换为**（仅保留幂等的索引创建）：
    ```js
    // 确保唯一索引存在（IF NOT EXISTS 保证幂等，不删除数据）
    db.exec(`
      CREATE UNIQUE INDEX IF NOT EXISTS idx_source_channels_user_url
      ON source_channels(user_id, url);
    `);
    ```
    
    注意：如果数据库中已有重复数据导致索引创建失败，这属于预先存在的数据问题，
    不应在每次启动时静默删除。索引创建失败会抛出错误，便于运维人员发现并手动处理。
  </action>
  <verify>node -e "const db = require('./db'); console.log('db 加载成功'); process.exit(0)"</verify>
  <done>
    1. db.js 中不再包含 `DELETE FROM source_channels` 字符串（grep 验证）
    2. `CREATE UNIQUE INDEX IF NOT EXISTS idx_source_channels_user_url` 仍然存在
    3. `node -e "require('./db')"` 在项目根目录执行成功，无报错
    4. 重复执行两次 require 不会删除任何数据（通过连续两次启动观察日志确认）
  </done>
