<objective>
修复 BUG-04：`checkHealth()` 通过正则解析远端日志文本（中文关键词 + `||` 行分隔）来判断任务状态，
任何 VPS locale 变化或日志格式微小改变都会导致所有状态判断静默失效。

改为**结构化 JSON 状态文件**方案：
- `buildCommand()` 生成的 bash 脚本在每个关键状态节点写出 `/tmp/restream_N.status` JSON 文件
- `checkHealth()` 通过 SSH 读取该 JSON 文件，解析后做精确判断，不依赖任何中文文本或正则

Purpose: 消除健康状态机对远端日志文本格式的脆弱依赖，让状态判断在任意 VPS 编码环境下稳定工作。
Output: 修改后的 task-manager.js，buildCommand 和 checkHealth 协同使用 JSON 状态文件。
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| VPS SSH | bash 脚本写出的 JSON 文件，通过 SSH cat 命令读取 |
| /tmp 目录 | VPS 上的临时文件，理论上可被同机其他进程修改 |
